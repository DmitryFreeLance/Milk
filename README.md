# MAX Milk Bot

Готовый бот для MAX на `Java + SQLite + Maven + Docker`, заточенный под приёмку молока по пунктам:

- `Чепас`
- `Большая Арать`
- `Пильна`

Стартовый справочник колхозов:

- `Березники`
- `Языково`
- `Байково`
- `Тарталей`

## Что умеет бот

- Роли: `администратор`, `директор`, `генеральный директор`, `сотрудник`
- Регистрация через заявку с ручным подтверждением администратором
- Назначение роли и пункта приёмки сотруднику
- Приёмка молока по шагам:
  - выбор колхоза
  - секция
  - вес
  - жир
  - белок
  - фото накладной
  - подтверждение
- Полная поддержка секций:
  - один поставщик в двух секциях оформляется двумя записями
  - аналитика автоматически суммирует общий вес
  - жир и белок считаются как взвешенные средние
- Расчёт зачётного веса по базису:
  - по умолчанию `жир 3.4%`
  - по умолчанию `белок 3.0%`
- Редактирование записи сотрудником в течение `1 часа`
- Повторное открытие записи администратором ещё на `1 час`
- Админ-панель:
  - заявки
  - пользователи
  - роли
  - пункты
  - колхозы
  - записи
  - удаление записей из активной базы
- Отчёты для руководителей:
  - карточка колхоза за отдельный день
  - сводка по пункту за период
  - сводка по всем пунктам за период
  - Excel-файл с графиками по среднему весу, жиру и белку
- Ежедневная сводка в конце смены по расписанию
- Все кнопки встроенные `inline`; короткие кнопки могут идти по 2 в ряд, длинные остаются по 1
- Бот не молчит:
  - на каждый сценарий есть ответ
  - при ошибке возвращает в безопасное меню
- Для доверенных тестовых пользователей из `BOOTSTRAP_ADMIN_USER_IDS` доступны slash-команды переключения ролей и пункта:
  - `/admin`
  - `/director`
  - `/gendirector`
  - `/employee`
  - `/user`
  - `/pending`
  - `/chepas`
  - `/arat`
  - `/pilna`
  - `/role`

## Архитектура

- `src/main/java/ru/milk/maxbot/App.java` — запуск приложения и long polling
- `config/AppConfig.java` — конфигурация через переменные окружения
- `db/Database.java` — создание схемы и сиды
- `repository/BotRepository.java` — работа с SQLite
- `max/MaxApiClient.java` — HTTP-клиент API MAX
- `service/BotService.java` — сценарии бота и роли
- `service/ReportService.java` — текстовые отчёты
- `service/ExcelReportService.java` — Excel и графики
- `service/SchedulerService.java` — автоматические сводки
- `service/PhotoService.java` — проверка фото и попытка автоповорота

## Настройка

Скопируйте значения из `.env.example` и задайте окружение:

```bash
export MAX_BOT_TOKEN="..."
export BOOTSTRAP_ADMIN_USER_IDS="123456789"
```

Первый администратор задаётся через `BOOTSTRAP_ADMIN_USER_IDS`.

Эти же пользователи могут использовать тестовые slash-команды для быстрого переключения своей роли и пункта приёмки без ручного редактирования через админку.

## Локальный запуск

```bash
mvn test
mvn package
java -jar target/max-milk-bot-1.0.0.jar
```

## Запуск в Docker

Сборка:

```bash
docker build -t max-milk-bot .
```

Запуск:

```bash
docker run -d \
  --name max-milk-bot \
  -e MAX_BOT_TOKEN="..." \
  -e BOOTSTRAP_ADMIN_USER_IDS="123456789" \
  -e MILK_BOT_TIMEZONE="Europe/Moscow" \
  -v $(pwd)/data:/app/data \
  max-milk-bot
```

## База данных

SQLite-файл по умолчанию:

```text
./data/milk-bot.sqlite
```

Основные таблицы:

- `users`
- `registration_requests`
- `receiving_points`
- `farms`
- `milk_receipts`
- `receipt_audit`
- `user_sessions`
- `daily_digest_log`

## Примечания по MAX

Проект использует `Long Polling`, потому что это было явно задано в задаче. Для production-деплоя сама платформа MAX рекомендует `Webhook`, поэтому архитектура сделана так, чтобы позже можно было без большого рефакторинга заменить источник обновлений.
